# 后台服务架构文档

## 1. 后台服务概述

DocSpace 后台服务由两大类组成：

1. **.NET 服务** - 运行在 `onlyoffice-dotnet-services` 容器中
2. **Java 服务** - 运行在 `onlyoffice-java-services` 容器中

这些服务由各自的 Supervisor 进行进程管理。

***

### 2.3 启动配置

所有 .NET 服务通过 Supervisor 管理，启动命令格式：

```bash
dotnet <服务名>.dll --urls=http://0.0.0.0:<端口> ...
```

关键特点：

- 绑定地址: `0.0.0.0`（监听所有接口）
- 自动重启: `autorestart=true`
- 启动延迟: `startsecs=25`

### 2.4 核心服务说明

#### ASC.Web.Api (端口 5000)

- 主API服务，处理所有 `/api/2.0/*` 请求
- 目录: `/var/www/services/ASC.Web.Api/service`
- 启动命令: `dotnet ASC.Web.Api.dll --urls=http://0.0.0.0:5000`

#### ASC.Files (端口 5007)

- 文件管理服务
- 目录: `/var/www/products/ASC.Files/server`

#### ASC.People (端口 5004)

- 人员/用户管理服务
- 目录: `/var/www/products/ASC.People/server`

## 3. Java 服务 (onlyoffice-java-services)

### 3.1 容器信息

- 容器名称: `onlyoffice-java-services`
- 镜像: `onlyoffice/docspace-java-services:latest`
- 进程管理: Supervisor

### 3.2 服务列表

| 服务名称                       | 端口   | 用途      | 状态      |
| -------------------------- | ---- | ------- | ------- |
| ASC.Identity.Authorization | 8080 | 身份授权服务  | RUNNING |
| ASC.Identity.Registration  | 9090 | 身份注册API | RUNNING |

### 3.3 启动配置

Java 服务使用 Spring Boot 框架：

```bash
java -jar <服务名>.jar --server.port=<端口>
```

关键环境变量：

- `SPRING_PROFILES_ACTIVE=dev,server`
- `SPRING_APPLICATION_ENCRYPTION_SECRET=...`
- `SPRING_APPLICATION_SIGNATURE_SECRET=...`

### 3.4 服务说明

#### ASC.Identity.Authorization (端口 8080)

- 负责用户身份认证和授权
- 处理 `/identity/*` 路由请求

#### ASC.Identity.Registration (端口 9090)

- 用户注册服务API
- 处理身份相关的REST API请求

## 5. 与 Router 的关联

### 5.1 Upstream 映射

Router 的 upstream.conf 定义了后台服务的地址映射：

```nginx
# .NET 服务
onlyoffice-dotnet-services:5000 → $service_api
onlyoffice-dotnet-services:5003 → $service_studio
onlyoffice-dotnet-services:5004 → $service_people_server
onlyoffice-dotnet-services:5007 → $service_files
onlyoffice-dotnet-services:5010 → $service_api_system
onlyoffice-dotnet-services:5012 → $service_backup
onlyoffice-dotnet-services:5033 → $service_healthchecks
onlyoffice-dotnet-services:5157 → $service_ai

# Java 服务
onlyoffice-java-services:8080 → $service_identity
onlyoffice-java-services:9090 → $service_identity_api
```

### 5.2 路由规则

| 路径             | 目标服务                              |
| -------------- | --------------------------------- |
| `/api/2.0/*`   | ASC.Web.Api (5000)                |
| `/people/*`    | ASC.People (5004)                 |
| `/files/*`     | ASC.Files (5007)                  |
| `/studio/*`    | ASC.Web.Studio (5003)             |
| `/backup/*`    | ASC.Data.Backup (5012)            |
| `/apisystem/*` | ASC.ApiSystem (5010)              |
| `/identity/*`  | ASC.Identity.Authorization (8080) |
| `/ai/*`        | ASC.AI (5157)                     |

## 7. 排查命令

### 7.1 查看 .NET 服务状态

```bash
docker exec onlyoffice-dotnet-services supervisorctl status
```

### 7.2 查看 Java 服务状态

```bash
docker exec onlyoffice-java-services supervisorctl status
```

### 7.3 查看服务日志

```bash
# .NET 服务日志
docker exec onlyoffice-dotnet-services cat /var/log/supervisor/ASC.Web.Api-stdout---supervisor-*.log

# Java 服务日志
docker exec onlyoffice-java-services cat /var/log/supervisor/ASC.Identity.Authorization-stdout---supervisor-*.log
```

### 7.4 测试服务连通性

```bash
# 从 Router 容器测试 .NET API
docker exec onlyoffice-router curl -I http://onlyoffice-dotnet-services:5000/

# 从 Router 容器测试 Java 身份服务
docker exec onlyoffice-router curl -I http://onlyoffice-java-services:8080/
```

# 前端服务架构文档

## 1. Node.js 微服务概述

OnlyOffice DocSpace 的前端服务由多个 Node.js 微服务组成，运行在 `onlyoffice-node-services` 容器中，由 Supervisor 进行进程管理。

### 1.1 微服务列表

| 服务名称 | 程序 | 目录 | 用途 |
|----------|------|------|------|
| ASC.Login | server.js | /var/www/products/ASC.Login/login | 用户登录认证 |
| ASC.Editors | server.js | /var/www/products/ASC.Editors/editor | 文档编辑器 |
| ASC.Management | server.js | /var/www/products/ASC.Management/management | 管理后台 |
| ASC.Sdk | server.js | /var/www/products/ASC.Sdk/sdk | SDK服务 |
| ASC.Socket.IO | server.js | /var/www/services/ASC.Socket.IO | WebSocket实时通信 |
| ASC.SsoAuth | app.js | /var/www/services/ASC.SsoAuth | SSO认证 |

## 2. 端口配置

### 2.1 容器内部端口

| 服务 | 端口 | 环境变量 | Nginx路由变量 |
|------|------|----------|---------------|
| Login | 5011 | SERVICE_LOGIN_PORT | $service_login |
| DocEditor | 5013 | SERVICE_DOCEDITOR_PORT | $service_doceditor |
| Management | 5015 | SERVICE_MANAGEMENT_PORT | $service_management |
| SDK | 5099 | SERVICE_SDK_PORT | $service_sdk |
| Socket.IO | 9899 | SERVICE_SOCKET_PORT | $service_socket |
| SsoAuth | 9834 | SERVICE_SSOAUTH_PORT | $service_sso |

### 2.2 Java服务端口（仅API缓存）

| 服务 | 端口 | 用途 |
|------|------|------|
| API Cache | 5100 | API响应缓存服务 |

### 2.3 容器间访问

在 Docker 内部网络中，各服务通过以下方式访问：

- 容器名: `onlyoffice-node-services`
- 完整地址: `onlyoffice-node-services:5011` (Login服务)
- Nginx upstream 配置使用容器名作为上游服务地址

### 2.3 宿主机访问

通过 Nginx 反向代理访问：
- 入口端口: 8092
- API端口: 8081

## 3. 服务启动配置

### 3.1 Supervisor 配置

服务通过 Supervisor 管理，配置文件位于 `/etc/supervisor/conf.d/supervisord.conf`（容器内）。

启动命令格式：
```bash
/usr/local/bin/node server.js --app.port=<端口> --app.appsettings=<配置文件路径> --app.environment=<环境类型>
```

### 3.2 环境变量

关键环境变量：
- `NODE_ENV=production`
- `PORT=<服务端口>`
- `SERVICE_LOGIN_PORT=5011`
- `SERVICE_DOCEDITOR_PORT=5013`
- `SERVICE_MANAGEMENT_PORT=5015`
- `SERVICE_SDK_PORT=5099`
- `SERVICE_SOCKET_PORT=9899`
- `SERVICE_SSOAUTH_PORT=9834`

## 4. Nginx 路由配置

### 4.1 Upstream 定义

Nginx 的 upstream 配置定义在 `/etc/nginx/conf.d/upstream.conf`：

```nginx
map onlyoffice-node-services:5011 $service_login {
    volatile;
    "" 127.0.0.1:5011;
    default onlyoffice-node-services:5011;
}
```

### 4.2 路由规则

| 路径 | 目标服务 | 端口 |
|------|----------|------|
| /login* | ASC.Login | 5011 |
| /api/2.0/authentication/* | ASC.Login | 5011 |
| /socket.io/* | ASC.Socket.IO | 9899 |
| /api/2.0/* | .NET API | 5000 |

## 5. 已知问题

### 5.1 服务监听IP问题

**问题描述**：Node.js 微服务启动时只监听容器内部IP地址，未监听 `127.0.0.1` 或 `0.0.0.0`。

**现象**：
- 从容器内 `127.0.0.1:5011` 访问失败
- 从其他容器通过 `onlyoffice-node-services:5011` 访问也失败
- Nginx 日志显示: `connect() failed (111: Connection refused)`

**证据**：
服务启动日志显示：
```
Starting server with environment: NODE_ENV=production, PORT=5011
   ▲ Next.js 16.0.10
   - Local:         http://b522090c7402:5011
   - Network:       http://b522090c7402:5011
```

可以看到服务只监听了容器内部 IP (`b522090c7402`)，没有监听 `0.0.0.0` 或 `127.0.0.1`。

**影响**：
- 容器间无法通过 `127.0.0.1` 访问服务
- Nginx 代理到服务时连接被拒绝
- 导致前端登录页面无法正常加载
- 所有需要访问 Node.js 微服务的 API 请求都会失败

**根本原因**：
Next.js 服务启动时未指定监听地址，默认只绑定到容器内部 IP。

**可能的解决方案**：
1. 修改服务启动命令，添加 `--app.host=0.0.0.0` 参数
2. 配置环境变量 `HOST=0.0.0.0`
3. 在 Supervisor 配置中添加监听地址参数

### 5.2 临时影响

由于上述问题，以下功能受影响：
- 用户登录页面无法正常显示
- 认证 API (authentication/status) 返回 502 错误
- 需要 Node.js 微服务的功能均无法正常使用

## 6. 排查命令

### 6.1 查看服务状态

```bash
# 查看 Supervisor 管理下的服务状态
docker exec onlyoffice-node-services supervisorctl status
```

### 6.2 查看服务日志

```bash
# 查看 ASC.Login 服务启动日志
docker exec onlyoffice-node-services cat /var/log/supervisor/ASC.Login-stdout---supervisor-*.log

# 查看服务错误日志
docker exec onlyoffice-node-services cat /var/log/supervisor/ASC.Login-stderr---supervisor-*.log
```

### 6.3 测试端口连通性

```bash
# 从容器内测试端口（目前会失败）
docker exec onlyoffice-node-services curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:5011/

# 从其他容器测试
docker exec onlyoffice-router curl -s -o /dev/null -w "%{http_code}" http://onlyoffice-node-services:5011/
```

### 6.4 查看 Nginx 错误日志

```bash
docker exec onlyoffice-router tail -f /var/log/nginx/error.log
```

## 7. 相关文件路径

- Supervisor配置: `/etc/supervisor/conf.d/supervisord.conf` (容器内)
- Nginx配置: `/etc/nginx/conf.d/onlyoffice.conf` (router容器内)
- Nginx upstream: `/etc/nginx/conf.d/upstream.conf` (router容器内)
- 前端静态文件: `/var/www/build/` (router容器内)

# DocSpace 密码算法与重置指南

## 1. 密码哈希算法概述

DocSpace 使用多层加密算法来保护用户密码，涉及 PBKDF2、SHA256、SHA512 等算法。

### 1.1 核心配置

密码哈希依赖以下配置项（位于 `appsettings.json`）：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `core:machinekey` | 机器密钥，用于生成 Salt 和最终哈希 | `DocSpace2024CoreMachineKeyForEncryption123!` |
| `core:password:size` | 哈希长度（位） | `256` |
| `core.password.iterations` | PBKDF2 迭代次数 | `100000` |
| `core:password:salt` | 自定义 Salt（可选） | 空（使用动态生成） |

### 1.2 源码位置

| 文件 | 说明 |
|------|------|
| `server/common/ASC.Common/Security/Cryptography/PasswordHasher.cs` | 密码哈希生成器 |
| `server/common/ASC.Common/Security/Cryptography/MachinePseudoKeys.cs` | 机器密钥提供者 |
| `server/common/ASC.Core.Common/Context/SecurityContext.cs` | 认证上下文 |
| `server/common/ASC.Core.Common/Data/DbUserService.cs` | 数据库用户服务 |

## 2. 密码哈希算法详解

### 2.1 算法流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                        密码哈希生成流程                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  输入: password (明文密码)                                       │
│        userId (用户ID)                                          │
│        machineKey (机器密钥)                                     │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 步骤1: 生成 Salt                                         │   │
│  │   saltSha256 = SHA256("{9450BEF7-7D9F-4E4F-A18A-971D8681722D}")│
│  │   saltBytes = PBKDF2(machineKey, saltSha256, SHA256, 100000, 32)│
│  │   salt = HexString(saltBytes)                            │   │
│  └─────────────────────────────────────────────────────────┘   │
│                           ↓                                     │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 步骤2: 生成客户端密码哈希 (GetClientPassword)             │   │
│  │   hashBytes = PBKDF2(password, salt, SHA256, 100000, 32) │   │
│  │   hexHash = HexString(hashBytes)                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                           ↓                                     │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 步骤3: 生成数据库存储哈希 (GetPasswordHash)               │   │
│  │   combined = hexHash + userId + machineKey               │   │
│  │   pwdHash = Base64(SHA512(combined))                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│                           ↓                                     │
│  输出: pwdHash (存储到 core_usersecurity.pwdhash)               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 步骤详解

#### 步骤1: 生成 Salt

Salt 是动态生成的，依赖于机器密钥：

```python
import hashlib

# 固定的种子字符串
SEED = "{9450BEF7-7D9F-4E4F-A18A-971D8681722D}"

# 计算 SHA256
salt_sha256 = hashlib.sha256(SEED.encode('utf-8')).digest()

# 使用 PBKDF2 派生 Salt
machine_key = "DocSpace2024CoreMachineKeyForEncryption123!"
salt_bytes = hashlib.pbkdf2_hmac(
    'sha256',
    machine_key.encode('utf-8'),  # password
    salt_sha256,                   # salt
    100000,                        # iterations
    32                             # dklen (256 bits / 8 = 32 bytes)
)
salt = salt_bytes.hex().lower()
```

**结果示例**:
```
salt = "8669f03825330f41afef4ed1057143162beff31764cf26db3121034958d34ff0"
```

#### 步骤2: 生成客户端密码哈希

使用 Salt 对用户密码进行哈希：

```python
password = "Admin@123"

hash_bytes = hashlib.pbkdf2_hmac(
    'sha256',
    password.encode('utf-8'),      # password
    salt.encode('utf-8'),           # salt (十六进制字符串)
    100000,                         # iterations
    32                              # dklen
)
hex_hash = hash_bytes.hex().lower()
```

**结果示例**:
```
hex_hash = "12baf10f08caeb458a21a6d4bcb8778ab722b1890f8515c85c6a3d2ea33dcc4c"
```

#### 步骤3: 生成数据库存储哈希

将客户端哈希与用户ID和机器密钥组合，生成最终存储的哈希：

```python
import base64

user_id = "66faa6e4-f133-11ea-b126-00ffeec8b4ef"

# 组合字符串
combined = hex_hash + user_id + machine_key

# SHA512 哈希并 Base64 编码
pwd_hash = base64.b64encode(
    hashlib.sha512(combined.encode('utf-8')).digest()
).decode('utf-8')
```

**结果示例**:
```
pwd_hash = "7q5zdrsdOKnE9WroIyODiaBcfsxAIbA7AT9itOtVj6R3RcpM+QuC6wVTVjRMlaj4U76KfnjGxRIud1oYVxyFSQ=="
```

## 5. 密码重置操作

### 5.1 使用 Python 脚本重置

使用 `reset_password.py` 脚本：

```bash
python reset_password.py
```

脚本会输出 SQL 更新语句，然后在数据库中执行。

### 5.2 手动重置步骤

1. **查询用户ID**:
```sql
SELECT id, username, email FROM core_user WHERE username='administrator';
```

2. **生成新密码哈希**:
   - 修改 `reset_password.py` 中的 `new_password` 变量
   - 运行脚本获取哈希值

3. **更新数据库**:
```sql
UPDATE core_usersecurity 
SET pwdhash='<新生成的哈希>' 
WHERE userid='<用户ID>';
```

### 5.3 重置后验证

```bash
# 测试登录
curl -X POST http://localhost:8092/api/2.0/authentication \
  -H "Content-Type: application/json" \
  -d '{"userName":"administrator","password":"Admin@123"}'
```

## 6. 安全注意事项

### 6.1 机器密钥保护

- `core:machinekey` 是密码系统的核心，必须妥善保管
- 生产环境应使用强随机密钥
- 密钥泄露会导致所有密码可被破解

### 6.2 密码策略

- 建议密码长度 >= 8 位
- 包含大小写字母、数字、特殊字符
- 定期更换密码

### 6.3 数据库安全

- 限制数据库访问权限
- 定期备份 `core_usersecurity` 表
- 记录密码修改审计日志

## 7. 常见问题

### Q1: 为什么密码哈希这么复杂？

多层加密设计是为了：
1. **防彩虹表攻击**: Salt 使每个系统的哈希不同
2. **防暴力破解**: PBKDF2 的 100000 次迭代增加计算成本
3. **防数据库泄露**: 即使数据库泄露，没有 machineKey 也无法破解

### Q2: 如何查看当前 machineKey？

```bash
# 查看容器内配置
docker exec onlyoffice-dotnet-services cat /app/onlyoffice/config/appsettings.json | grep machinekey
```

### Q3: 重置密码后还是无法登录？

检查以下项：
1. 用户状态是否正常 (`status=0`)
2. 服务是否正常运行
3. 清除浏览器缓存和 Cookie

# 文档服务前端配置流程

## 1. 概述

DocSpace 设置界面中「文档服务」配置的前端提交流程。

## 2. 涉及的容器和服务

| 容器名称 | 服务 | 端口 | 职责 |
|----------|------|------|------|
| `onlyoffice-router` | Nginx | 8092 | 反向代理，接收前端请求并路由到后端 |
| `onlyoffice-dotnet-services` | ASC.Web.Api | 5000 | 处理 `PUT /api/2.0/files/docservice` 请求，保存配置到数据库 |
| `onlyoffice-document-server` | DocumentServer | 80 (内部) / 8085 (映射) | 被配置的文档编辑服务 |

## 3. 请求完整路径

```
浏览器
    ↓ PUT http://localhost:8092/api/2.0/files/docservice
onlyoffice-router:8092 (Nginx)
    ↓ proxy_pass to onlyoffice-dotnet-services:5000
onlyoffice-dotnet-services:5000 (ASC.Web.Api)
    ↓ 处理并保存配置
数据库
```

## 4. 端口说明

| 访问方式 | 地址 | 说明 |
|----------|------|------|
| 宿主机访问 | `http://localhost:8092` | Web 入口，Nginx 监听端口 |
| 宿主机访问 | `http://localhost:8085` | DocumentServer 端口映射 |
| 容器间访问 | `http://onlyoffice-dotnet-services:5000` | .NET API 服务地址 |
| 容器间访问 | `http://onlyoffice-document-server:80` | DocumentServer 容器内部地址 |

## 5. 详细代码路径

### 5.1 前端设置界面组件

**文件**: `client/packages/client/src/pages/PortalSettings/categories/integration/DocumentService/index.js`

**关键函数**:
- `onSubmit` (第142行) - 表单提交处理函数
- `changeDocumentServiceLocation(...)` (第146-153行) - 调用 API 函数

**提交参数**:
```javascript
changeDocumentServiceLocation(
  docServiceUrl,        // DocumentService URL (如 http://localhost:8085/)
  secretKey,            // JWT 密钥
  authHeader,            // 授权头 (AuthorizationJwt)
  internalUrl,          // 内部 URL
  portalUrl,            // Portal URL (如 http://localhost:8092/)
  !isDisabledCertificat // SSL 验证开关
)
```

### 5.2 API 函数定义

**文件**: `client/packages/shared/api/files/index.ts`

```typescript
export async function changeDocumentServiceLocation(
  docServiceUrl: string,
  secretKey: string,
  authHeader: string,
  internalUrl: string,
  portalUrl: string,
  sslVerification: boolean,
) {
  const res = await request({
    method: "put",
    url: `files/docservice`,
    data: {
      DocServiceUrl: docServiceUrl,
      DocServiceUrlInternal: internalUrl,
      DocServiceUrlPortal: portalUrl,
      DocServiceSignatureSecret: secretKey,
      DocServiceSignatureHeader: authHeader,
      DocServiceSslVerification: sslVerification,
    },
  });
  return res;
}
```

### 5.3 后端处理 Controller

**文件**: `server/products/ASC.Files/Server/Api/EditorController.cs`

**处理方法**: `CheckDocServiceUrl` (第350行)
- **路由**: `PUT /api/2.0/files/docservice`
- **权限**: 需要 `EditPortalSettings` 权限

**处理逻辑**:
1. 验证 URL 格式
2. 保存配置到数据库
3. 调用 `documentServiceConnector.CheckDocServiceUrlAsync()` 验证连接
4. 返回更新后的配置

## 6. 错误排查

### 6.1 500 Internal Server Error

**可能原因**:
1. DocumentServer 未运行或无法连接
2. JWT 密钥不匹配
3. URL 格式验证失败
4. `documentServiceConnector.CheckDocServiceUrlAsync()` 调用失败

**排查命令**:
```bash
# 检查 DocumentServer 状态
docker exec onlyoffice-document-server supervisorctl status

# 从 .NET 容器测试 DocumentServer 连通性
docker exec onlyoffice-dotnet-services curl -I http://onlyoffice-document-server:80/
```

### 6.2 保存配置时回滚

后端代码第398-408行显示，如果 `CheckDocServiceUrlAsync()` 抛出异常，配置会自动回滚到之前的值。这意味着保存失败后会恢复原状。

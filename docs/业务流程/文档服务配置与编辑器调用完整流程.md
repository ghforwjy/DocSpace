# 文档服务配置与编辑器调用完整流程

## 1. 概述

本文档完整描述 DocSpace 系统中 **文档服务的配置流程** 和 **文档编辑器调用流程**，涵盖从管理员配置 DocumentServer，到用户打开文档进行编辑的完整链路。

---

## 4. 流程一：文档服务配置

### 4.1 配置概述

管理员在 DocSpace 设置界面中配置 DocumentServer 的连接信息，包括：
- DocumentServer URL（对外访问地址）
- 内部 URL（容器间访问地址）
- JWT 密钥（用于请求签名验证）
- 授权头名称
- SSL 验证开关

### 4.2 配置请求路径

```
用户浏览器
    ↓ PUT http://localhost:8092/api/2.0/files/docservice
onlyoffice-router:8092 (Nginx)
    ↓ proxy_pass → onlyoffice-dotnet-services:5000
onlyoffice-dotnet-services:5000 (ASC.Web.Api)
    ↓ 验证 URL 格式
    ↓ 保存配置到数据库
    ↓ 调用 CheckDocServiceUrlAsync() 验证连接
数据库 / DocumentServer
```

### 4.3 前端配置界面

**前端组件**: `client/packages/client/src/pages/PortalSettings/categories/integration/DocumentService/index.js`

**关键函数**:
- `onSubmit` (第142行) - 表单提交处理函数
- `changeDocumentServiceLocation(...)` (第146-153行) - 调用 API 函数

**提交参数**:

| 前端界面字段名 | 参数名 | 说明 | 示例值 |
|--------------|--------|------|--------|
| **文件编辑服务地址** | `docServiceUrl` | DocumentServer URL | `http://localhost:8085/` |
| **密钥** | `secretKey` | JWT 密钥 | `DocSpace2024SecureJwtSecretKey123!` |
| **授权标头** | `authHeader` | 授权头名称 | `AuthorizationJwt` |
| **来自协作空间的请求的文档服务地址** | `internalUrl` | 内部 URL | `http://onlyoffice-document-server:80/` |
| **来自文档服务的请求的协作空间地址** | `portalUrl` | Portal URL | `http://localhost:8092/` |
| **禁用证书验证（不安全）** | `isDisabledCertificat` | SSL 验证开关 | ✅ 勾选 |

### 4.4 配置参数填写规则与示例

> **重要**：DocumentServer 与 DocSpace 在同一 Docker 网络中部署时，**来自文档服务的请求的协作空间地址** 不能使用 `localhost`。

#### 4.4.1 网络上下文说明

| 上下文 | localhost 指向 | 说明 |
|--------|---------------|------|
| 宿主机浏览器 | 宿主机本身 | 用户访问 DocSpace 的入口 |
| DocumentServer 容器内 | DocumentServer 容器本身 | 无法访问 DocSpace 容器 |

**结论**：`localhost` 在 DocumentServer 容器内无法访问 DocSpace，因此 `portalUrl` 不能填 `localhost`。

#### 4.4.2 同一 Docker 网络下的正确配置

> **⚠️ 重要说明**：
> - `localhost` 在 **.NET 容器内指向容器自己**，无法访问其他容器
> - 因此 **"文件编辑服务地址"** 不能填 `localhost`，必须填 Docker 网络地址
>
> **HTTPS 配置**：如果需要使用 HTTPS 连接 DocumentServer，请参考 [4.4.7 节](#447-documentserver-httpsssl-证书配置) 先在 DocumentServer 容器内启用 SSL。

当 DocumentServer 与 DocSpace 在同一 Docker 网络中时：

| 前端界面字段名 | HTTP 配置 | HTTPS 配置 | 说明 |
|--------------|-----------|------------|------|
| **文件编辑服务地址** | `http://onlyoffice-document-server:80/` | `https://onlyoffice-document-server:443/` | .NET 后端访问 DocumentServer 用 |
| **密钥** | `DocSpace2024SecureJwtSecretKey123!` | 同左 | 必须与 DocumentServer 的 `JWT_SECRET` 一致 |
| **授权标头** | `AuthorizationJwt` | 同左 | 必须与 DocumentServer 的 `JWT_HEADER` 一致 |
| **来自协作空间的请求的文档服务地址** | `http://onlyoffice-document-server:80/` | `https://onlyoffice-document-server:443/` | 同上 |
| **来自文档服务的请求的协作空间地址** | `http://onlyoffice-dotnet-services:5000/` | 同左 | DocumentServer 回调 DocSpace 用 |
| **禁用证书验证（不安全）** | ✅ 勾选 | **不勾选** | HTTP 用自签名证书勾选；HTTPS 不勾选 |

#### 4.4.3 网络上下文说明

| 上下文 | localhost 指向 | 说明 |
|--------|---------------|------|
| 宿主机浏览器 | 宿主机本身 | 用户访问 DocSpace 的入口 |
| .NET 容器内 | .NET 容器本身 | 无法访问 DocumentServer |
| DocumentServer 容器内 | DocumentServer 容器本身 | 无法访问 DocSpace |

**结论**：
- 在 Docker 网络中，**不要使用 `localhost`**
- 所有容器间通信必须使用 **Docker 网络地址**（如 `onlyoffice-document-server:80`）

### 4.4.7 DocumentServer HTTPS/SSL 证书配置

> **重要**：如果使用 HTTPS 连接 DocumentServer，必须先在 DocumentServer 容器内启用 SSL 并配置证书。

#### 4.4.7.1 检查当前 SSL 状态

默认情况下 DocumentServer 的 nginx 只监听 80 端口，443 端口未启用：

```bash
# 检查 DocumentServer 端口监听状态
wsl docker exec onlyoffice-document-server ss -tlnp

# 正常情况（仅 HTTP）：
# 0.0.0.0:80        - nginx HTTP
# *:8000             - docservice

# 启用 HTTPS 后应该有：
# 0.0.0.0:443        - nginx HTTPS
# 0.0.0.0:80         - nginx HTTP (重定向到 HTTPS)
```

#### 4.4.7.2 生成自签名 SSL 证书

在 DocumentServer 容器内生成自签名证书：

```bash
# 1. 创建 SSL 证书目录
wsl docker exec onlyoffice-document-server mkdir -p /etc/onlyoffice/documentserver/ssl

# 2. 生成自签名证书（有效期 365 天）
wsl docker exec onlyoffice-document-server openssl req -x509 -nodes -days 365 \
  -newkey rsa:2048 \
  -keyout /etc/onlyoffice/documentserver/ssl/self.key \
  -out /etc/onlyoffice/documentserver/ssl/self.crt \
  -subj '/CN=onlyoffice-document-server'
```

#### 4.4.7.3 配置 nginx SSL

创建 SSL nginx 配置文件（复制到本地编辑后上传）：

**本地文件**: `<项目根目录>/temp/ds-ssl.conf`

```nginx
include /etc/nginx/includes/http-common.conf;

server {
  listen 0.0.0.0:80;
  listen [::]:80 default_server;
  server_name _;
  server_tokens off;
  set $secure_link_secret verysecretstring;
  root /nowhere;
  rewrite ^ https://$host$request_uri? permanent;
}

server {
  listen 127.0.0.1:80;
  listen [::1]:80;
  server_name localhost;
  server_tokens off;
  set $secure_link_secret verysecretstring;
  include /etc/nginx/includes/ds-common.conf;
  include /etc/nginx/includes/ds-docservice.conf;
}

server {
  listen 0.0.0.0:443 ssl http2;
  listen [::]:443 ssl default_server http2;
  server_tokens off;
  set $secure_link_secret verysecretstring;
  root /usr/share/nginx/html;
  ssl_certificate /etc/onlyoffice/documentserver/ssl/self.crt;
  ssl_certificate_key /etc/onlyoffice/documentserver/ssl/self.key;
  ssl_verify_client off;
  ssl_ciphers "EECDH+AESGCM:EDH+AESGCM:AES256+EECDH:AES256+EDH";
  ssl_protocols TLSv1.2;
  ssl_session_cache builtin:1000 shared:SSL:10m;
  ssl_prefer_server_ciphers on;
  add_header Strict-Transport-Security max-age=31536000;
  add_header X-Content-Type-Options nosniff;
  include /etc/nginx/includes/ds-*.conf;
}
```

#### 4.4.7.4 上传配置并重启 nginx

```bash
# 1. 上传 SSL 配置文件
cat <项目根目录在WSL中的路径>/temp/ds-ssl.conf | wsl docker exec -i onlyoffice-document-server tee /etc/nginx/conf.d/ds-ssl.conf

# 2. 删除原有的 HTTP-only 配置
wsl docker exec onlyoffice-document-server rm -f /etc/nginx/conf.d/ds.conf

# 3. 测试 nginx 配置
wsl docker exec onlyoffice-document-server nginx -t

# 4. 重载 nginx
wsl docker exec onlyoffice-document-server nginx -s reload
```

#### 4.4.7.5 验证 HTTPS 启用

```bash
wsl docker exec onlyoffice-document-server ss -tlnp
```

### 4.5 API 函数定义

**文件**: `client/packages/shared/api/files/index.ts`

```typescript
export async function changeDocumentServiceLocation(
  docServiceUrl: string,
  secretKey: string,
  authHeader: string,
  internalUrl: string,
  portalUrl: string,
  sslVerification: boolean,
) {
  const res = await request({
    method: "put",
    url: `files/docservice`,
    data: {
      DocServiceUrl: docServiceUrl,
      DocServiceUrlInternal: internalUrl,
      DocServiceUrlPortal: portalUrl,
      DocServiceSignatureSecret: secretKey,
      DocServiceSignatureHeader: authHeader,
      DocServiceSslVerification: sslVerification,
    },
  });
  return res;
}
```

### 4.6 后端处理

**文件**: `server/products/ASC.Files/Server/Api/EditorController.cs`

**处理方法**: `CheckDocServiceUrl` (第350行)

**路由**: `PUT /api/2.0/files/docservice`

**权限**: 需要 `EditPortalSettings` 权限

**处理逻辑**:
1. 验证 URL 格式
2. 保存配置到数据库

#### 步骤4：getData API 调用

**文件**: `client/packages/doceditor/src/utils/actions.ts`

```typescript
export async function getData(
    fileId: string,
    version?: string,
    doc?: string,
    action?: ActionType,
    share?: string,
    editorType?: string,
) {
    const view = action === "view";

    const searchParams = new URLSearchParams();
    if (action && availableActions[action]) searchParams.append(action, "true");
    if (version) searchParams.append("version", version);
    if (doc) searchParams.append("doc", doc);
    if (share) searchParams.append("share", share);
    if (editorType) searchParams.append("editorType", editorType);

    // 并行调用三个API
    const [config, user, settings] = await Promise.all([
        openEdit(fileId, searchParams.toString(), share),  // 获取编辑器配置
        getUser(share),                                    // 获取用户信息
        getSettings(share),                                // 获取系统设置
    ]);

    return { config, user, settings, successAuth: !!user };
}
```

#### 步骤5：openEdit 调用后端 API

```typescript
export async function openEdit(

    searchParams: string,
    share?: string,
) {
    // 调用 GET /api/2.0/files/file/{fileId}/openedit
    const [getConfig] = await createRequest(
        [`/files/file/${fileId}/openedit?${searchParams}`],
        [share ? ["Request-Token", share] : ["", ""]],
        "GET",
        undefined,
    );

    const res = await fetch(getConfig);
    const config = await res.json();

    if (res.ok) {
        config.response.editorUrl = config.response.editorUrl.replace(REPLACED_URL_PATH, "");
        return { ...config.response };
    }

    return error;
}
```

```csharp
public async Task<(File<T> File, Configuration<T> Configuration, bool LocatedInPrivateRoom)> GetParamsAsync<T>(
    T fileId,
    int version,
    bool editPossible,
    bool tryEdit,

{
```

## 12. 已知问题

### 12.1 Node.js 服务监听地址问题 ✅ 已修复

**问题描述**：Router 将请求代理到 Node.js 服务（如 ASC.Login），但由于 Node.js 服务只监听容器内部IP，未监听 `127.0.0.1` 或 `0.0.0.0`，导致代理连接失败。

**现象**：
- 访问 `/login` 时返回 502 Bad Gateway 或 504 Gateway Timeout
- Nginx 错误日志：`connect() failed (111: Connection refused)`

**受影响的路由**：
- `/login/*` → ASC.Login (5011)
- `/doceditor/*` → ASC.Editors (5013)
- `/management/*` → ASC.Management (5015)
- `/sdk/*` → ASC.Sdk (5099)
- `/socket.io/*` → ASC.Socket.IO (9899)
- `/sso/*` → ASC.SsoAuth (9834)

**修复方案**：在 Supervisor 配置中添加 `--app.host=0.0.0.0` 参数。

### 12.2 SSR 时 API 请求失败（API_HOST 缺失） ✅ 已修复

**问题描述**：SSR（服务端渲染）时，Node.js 内部调用 API 失败，导致登录页面显示空白。

**现象**：
- 访问 `/login` 时页面显示空白背景，没有登录控件
- 浏览器控制台显示 `wrongPortalNameUrl: ""`
- Node.js 容器日志显示 `Error in getSettings: TypeError: fetch failed`

**修复方案**：在 Supervisor 配置中添加 `API_HOST` 环境变量。

### 12.3 ASC.Socket.IO 和 ASC.SsoAuth 服务 FATAL 问题 ✅ 已修复

**问题描述**：
- ASC.Socket.IO 和 ASC.SsoAuth 服务处于 FATAL 状态
- 错误日志：`Cannot read properties of undefined (reading 'cloudWatch')`

**修复方案**：
- 修正 `PATH_TO_CONF` 为目录路径而非文件路径
- 添加 `--app.host=0.0.0.0` 参数

### 12.4 保存文档服务配置 500 错误

**可能原因**：
1. DocumentServer 未运行或无法连接
2. JWT 密钥不匹配
3. URL 格式验证失败
4. `CheckDocServiceUrlAsync()` 调用失败

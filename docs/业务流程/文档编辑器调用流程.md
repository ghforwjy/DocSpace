# 文档编辑器调用流程

## 1. 整体架构

DocSpace 系统打开文档涉及以下组件：

| 组件 | 容器 | 端口 | 技术栈 | 说明 |
|------|------|------|--------|------|
| Router | onlyoffice-router | 8092 | Nginx | 反向代理入口 |
| ASC.Editors | onlyoffice-node-services | 5013 | Node.js/Next.js | 文档编辑器前端 |
| ASC.Web.Api | onlyoffice-dotnet-services | 5000 | .NET | 主API服务 |
| ASC.Files | onlyoffice-dotnet-services | 5007 | .NET | 文件管理服务 |
| ASC.People | onlyoffice-dotnet-services | 5004 | .NET | 用户管理服务 |
| ASC.Identity | onlyoffice-java-services | 8080 | Java/Spring | 身份授权服务 |
| Document Server | onlyoffice-documentserver | 80/443 | Node.js | 文档转换渲染 |

## 4. 详细流程步骤

### 步骤1：用户访问入口

```
URL: http://domain:8092/doceditor?fileId=2
```

用户浏览器访问 DocSpace 文档编辑器入口地址。

### 步骤2：Router代理到Editor服务

Router 根据 upstream.conf 配置将请求代理到 Node.js 容器：

```nginx
# upstream.conf
map onlyoffice-node-services:5013 $service_doceditor

# onlyoffice.conf
location /doceditor {
    proxy_pass http://$service_doceditor;
}
```

**目标地址**: `onlyoffice-node-services:5013`

### 步骤3：Node.js编辑器服务处理

**文件**: `client/packages/doceditor/src/app/(root)/page.tsx`

```typescript
async function Page(props: RootPageProps) {
    const { searchParams: sp } = props;

```

### 步骤4：getData API调用

**文件**: `client/packages/doceditor/src/utils/actions.ts`

```typescript
export async function getData(
    fileId: string,
    version?: string,
    doc?: string,
    action?: ActionType,
    share?: string,
    editorType?: string,
) {
    const view = action === "view";

    const searchParams = new URLSearchParams();
    if (action && availableActions[action]) searchParams.append(action, "true");
    if (version) searchParams.append("version", version);
    if (doc) searchParams.append("doc", doc);
    if (share) searchParams.append("share", share);
    if (editorType) searchParams.append("editorType", editorType);

    // 并行调用三个API
    const [config, user, settings] = await Promise.all([
        openEdit(fileId, searchParams.toString(), share),  // 获取编辑器配置
        getUser(share),                                    // 获取用户信息
        getSettings(share),                                // 获取系统设置
    ]);

    return { config, user, settings, successAuth: !!user };
}
```

### 步骤5：openEdit调用后端API

```typescript
export async function openEdit(
    fileId: number | string,
    searchParams: string,
    share?: string,
) {
    // 调用 GET /api/2.0/files/file/{fileId}/openedit
    const [getConfig] = await createRequest(
        [`/files/file/${fileId}/openedit?${searchParams}`],
        [share ? ["Request-Token", share] : ["", ""]],
        "GET",
        undefined,
    );

    const res = await fetch(getConfig);
    const config = await res.json();

    if (res.ok) {
        // 处理返回的配置
        config.response.editorUrl = config.response.editorUrl.replace(REPLACED_URL_PATH, "");
        return { ...config.response };
    }

    // 处理错误情况
    return error;
}
```

### 步骤6：后端EditorController处理

**文件**: `server/products/ASC.Files/Server/Api/EditorController.cs`

```csharp
[HttpGet("{fileId}/openedit")]
public async Task<ConfigurationDto<T>> OpenEditFile(OpenEditRequestDto<T> inDto)
{
    // 1. 获取文件信息
    var (file, lastVersion) = await documentServiceHelper.GetCurFileInfoAsync(
        inDto.FileId,
        inDto.Version
    );

    // 2. 获取编辑器配置参数
    var docParams = await documentServiceHelper.GetParamsAsync(
        file,
        lastVersion,
        await documentServiceHelper.CheckCustomQuota(rootFolder),
        !inDto.View,
        true,
        formOpenSetup == null || formOpenSetup.CanFill,
        formOpenSetup?.EditorType ?? inDto.EditorType,
        formOpenSetup is { IsSubmitOnly: true }
    );

    // 3. 返回配置
    var configuration = docParams.Configuration;
    return await configurationConverter.Convert(configuration, file);
}
```

### 步骤7：DocumentServiceHelper生成配置

**文件**: `server/products/ASC.Files/Core/Services/DocumentService/DocumentServiceHelper.cs`

```csharp
public async Task<(File<T> File, Configuration<T> Configuration, bool LocatedInPrivateRoom)> GetParamsAsync<T>(
    T fileId,
    int version,
    bool editPossible,
    bool tryEdit,
    bool tryCoAuthoring,
    bool fillFormsPossible
)
{
    // 1. 获取文件信息
    var (file, lastVersion) = await GetCurFileInfoAsync(fileId, version);

    // 2. 检查用户权限
    var rightToEdit = rightToEdit && (await fileSecurity.CanEditAsync(file));
    // ... 其他权限检查

    // 3. 生成文档唯一key (用于Document Server缓存)
    var docKey = DocumentService.GenerateRevisionId(...);

    // 4. 获取文档下载地址
    var docUri = await fileStorageService.GetPreSignedUriAsync(file.Id);

    // 5. 构建配置
    Configuration = new Configuration<T>
    {
        Document = new Document<T>
        {
            Key = docKey,
            Url = docUri,
            Title = file.Title,
            FileType = file.Exst
        },
        EditorConfig = editorConfig,
        EditorUrl = GetDocServiceUrl()  // Document Server 地址
    };

    return (file, Configuration, LocatedInPrivateRoom);
}
```

### 步骤8：Document Server渲染编辑器

Document Server 收到配置后:

1. 根据 `document.key` 判断文档是否已缓存
2. 如需下载，从 `document.url` 下载文档
3. 调用 `DocEditor` SDK 渲染编辑器界面

## 8.5 控制台配置输出

文档编辑器页面在浏览器控制台会输出两部分内容：

### 输出 1：`It's DocEditor INIT` (Scripts.tsx)

**来源**: `fetch("/static/scripts/config.json")` 的响应

**作用**: 确认 CSR 客户端配置是否正确加载

### 输出 2：`initCSR` (axiosClient.ts)

**来源**: `axiosClient.initCSR()` 初始化参数

**作用**: 确认 axios 初始化参数是否正确

## 8.6 配置文件路径与访问方式

DocEditor 存在两套配置文件加载路径，取决于用户如何访问页面：

### 方式一：通过 Router 访问 (`http://localhost:8085/doceditor`)

```
用户 → Router (8085) → Nginx → Node DocEditor (5013)
```

| 阶段 | 配置文件路径 | 说明 |
|------|-------------|------|
| 编译时 | `client/public/scripts/config.json` | 构建时打包到镜像 |
| 运行时 | `Router 容器内: /var/www/client/static/scripts/config.json` | Nginx root 目录 |
| 请求 URL | `http://localhost:8085/static/scripts/config.json` | 用户浏览器实际请求 |

**Nginx 路由规则** (onlyoffice.conf):
```nginx
location ~* /static/scripts/(.*)$ {
    root /var/www/client;  # root 指向 /var/www/client
    try_files /scripts/$1 /index.html =404;
}
# 实际映射: /var/www/client/static/scripts/config.json
```

### 方式二：直连 Node 服务访问 (`http://localhost:5013/doceditor`)

```
用户 → Node DocEditor (5013) 直接访问
```

| 阶段 | 配置文件路径 | 说明 |
|------|-------------|------|
| 编译时 | `client/public/scripts/config.json` | 构建时打包到镜像 |
| 运行时 | `Node 容器内: /app/static/scripts/config.json` | Next.js publicDir |
| 请求 URL | `http://localhost:5013/static/scripts/config.json` | 用户浏览器实际请求 |

**Next.js 配置**:
- `dev`: `publicDir: ./public`
- `start-prod`: `root: /app`

### 关键差异

| 访问方式 | 配置文件在容器内位置 | 请求路径 |
|---------|---------------------|---------|
| Router (8085) | `/var/www/client/static/scripts/config.json` | `/static/scripts/config.json` |
| 直连 Node (5013) | `/app/static/scripts/config.json` | `/static/scripts/config.json` |

**注意**: 两种方式的请求 URL 相同 (`/static/scripts/config.json`)，但容器内的实际路径不同。

### 配置来源

`config.json` 编译时内容来自 `client/packages/shared/utils/axiosClient.ts`:

```typescript
import defaultConfig from "PUBLIC_DIR/scripts/config.json";  // 编译时嵌入
```

构建后镜像中的 `config.json` 实际来源:
- 源码: `client/public/scripts/config.json`
- 构建时复制到镜像的 public 目录

### 诊断建议

1. **通过 Router 访问失败时**: 检查 Router 容器内 `/var/www/client/static/scripts/config.json` 是否存在
2. **直连 Node 访问失败时**: 检查 Node 容器内 `/app/static/scripts/config.json` 是否存在
3. **两边都不对**: 检查 `client/public/scripts/config.json` 源码内容

### initCSR 输出参数

| 参数 | 含义 | 诊断价值 |
|------|------|----------|
| `defaultConfig` | 编译时嵌入的配置 | 对比运行时配置是否一致 |
| `apxiosConfig.baseURL` | axios 请求基础URL | ⚠️ **如果与预期不符，说明 ClientConfig 或 defaultConfig 有问题** |
| `apxiosConfig.timeout` | axios 超时 | 确认是否使用配置值 |
| `ClientConfig` | 运行时加载的配置 | 与 `defaultConfig` 对比 |
| `paymentsURL` | 支付页面URL | 确认构造是否正确 |

### 诊断流程

当文档编辑器页面出现问题时，按以下顺序检查：

```
1. 检查控制台是否有输出
   ├── 有 "It's DocEditor INIT" 输出 → CSR 脚本执行正常
   ├── 无输出 → fetch config.json 失败，检查 Router 日志
   │
2. 检查 initCSR 输出
   ├── apxiosConfig.baseURL 是否正确
   ├── ClientConfig 与 defaultConfig 是否一致
   │
3. 检查 API 请求（浏览器 Network 面板）
   ├── /files/file/{id}/openedit 是否返回 200
   ├── 响应内容是否是有效的配置对象
   │
4. 如果配置返回错误
   ├── 404 → file-not-found，检查 fileId 参数
   ├── 403 → access-denied，权限问题
   └── 其他 → 检查 .NET Services 是否正常运行
```

## 9. 渲染模式

文档编辑器采用 **CSR (客户端渲染)** 模式:

1. **SSR阶段**: Node.js 服务端负责渲染页面框架
2. **API调用**: 前端调用后端API获取编辑器配置
3. **SDK加载**: 浏览器加载 DocumentServer 的 api.js
4. **客户端渲染**: 使用返回的配置初始化 DocEditor
5. **实时协作**: Document Server 通过 WebSocket (`/socket.io`) 支持多人协作编辑

### 问题2：SSR时API_HOST缺失

**现象**: 访问 `/doceditor` 时，如果使用SSR调用后端API可能失败

**原因**:
- SSR时 `getAPIUrl()` 使用 `x-forwarded-host` header
- 容器内 `localhost:8092` 指向容器自己，不是Router

**解决**: 在 Supervisor 配置中添加 `API_HOST` 环境变量

### 问题3：Document Server JWT验证

**注意**: Document Server 默认不启用JWT验证

**生产环境建议**: 启用JWT验证以确保安全性

```yaml
environment:
  - JWT_ENABLED=true
  - JWT_SECRET=your-secret-key
```

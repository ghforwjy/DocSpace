
**影响：** Java 服务的消息队列功能不可用，可能导致用户认证/注册功能异常

## 九、根因分析

### Java 服务 RabbitMQ 连接失败

从 `.env` 文件分析：
- `RABBIT_HOST=` (空)
- `RABBIT_CONTAINER_NAME=onlyoffice-rabbitmq`

在 `docspace-stack.yml` 中：
```yaml
RABBIT_HOST: ${RABBIT_CONTAINER_NAME}  # 应该是 onlyoffice-rabbitmq
```

但 Java 服务的 Spring Boot 应用可能读取的是 `RABBIT_URI` 或其他变量，导致连接失败。

**建议检查：**
1. 验证 Java 服务的启动脚本是否正确设置 RabbitMQ 相关的环境变量
2. 检查 `docker-entrypoint.sh` 是否正确处理 RabbitMQ 配置
3. 可能需要为 Java 服务单独设置 `RABBIT_URI` 环境变量

## 十、修复建议

### 1. 修复 Java 服务 RabbitMQ 连接

需要在 `.env` 文件中添加：
```
RABBIT_URI=amqp://guest:guest@onlyoffice-rabbitmq:5672/
```

或在 `docspace-stack.yml` 中确保正确传递 `RABBIT_HOST` 和 `RABBIT_URI` 环境变量。

### 2. 修复 Document Server 启动问题

**可能的问题原因：**
- pluginsmanager 进程卡住（CPU 99%）
- 启动脚本在某处等待超时

**修复步骤：**
```bash
# 重启 Document Server 容器
docker restart onlyoffice-document-server

# 如果问题依旧，检查插件配置
# 查看完整启动日志
docker logs onlyoffice-document-server --tail 200
```

**临时解决（如果不需要文档编辑功能）：**
Document Server 容器可以保持当前状态，不影响 DocSpace 核心功能（用户管理、房间管理等）。

### 3. 检查数据库迁移

```bash
# 重新运行迁移
docker compose -f migration-runner.yml run --rm onlyoffice-migration-runner
```

### 4. 添加 Document Server（如需要文档编辑功能）

```bash
# 确保 Document Server 正确启动
docker compose -f ds.yml up -d

# 检查启动状态
docker logs onlyoffice-document-server --follow
```

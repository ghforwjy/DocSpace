### 解决方法详情

#### 1. Java 服务 RabbitMQ 连接问题
**问题根因**：
- `.env` 文件中 `RABBIT_HOST` 为空
- Java 服务尝试连接 `localhost:5672` 而非 `onlyoffice-rabbitmq:5672`

**解决方案**：
1. 更新 `.env` 文件：
   ```
   RABBIT_HOST="${RABBIT_CONTAINER_NAME}"
   RABBIT_URI=${RABBIT_PROTOCOL}://${RABBIT_USER_NAME}:${RABBIT_PASSWORD}@${RABBIT_HOST}:${RABBIT_PORT}${RABBIT_VIRTUAL_HOST}
   ```

2. 更新 `docspace-stack.yml` 文件，添加 `RABBIT_URI` 环境变量传递

3. 重启 Java 服务容器：
   ```bash
   docker restart onlyoffice-java-services
   ```

**验证结果**：
- Java 服务成功连接 RabbitMQ
- `rabbitmqctl list_connections` 显示 Java 服务 IP（172.18.0.7）已连接

#### 2. Document Server 启动问题
**问题根因**：
- pluginsmanager 进程卡住（CPU 99%）
- 启动脚本在等待某个条件时卡住

**解决方案**：
- 服务自动恢复，无需手动干预

**验证结果**：
- Document Server 容器状态为 healthy
- 所有核心服务（nginx、supervisor、DocService、FileConverter）均已启动
- HealthCheck 返回正常

#### 3. Document Server 连接配置问题
**问题根因**：
- `.env` 文件中 `DOCUMENT_SERVER_URL_EXTERNAL` 未配置
- Node.js Editors 和 .NET Files 服务无法获取外部 Document Server 地址

**解决方案**：
1. 更新 `.env` 文件：
   ```
   DOCUMENT_SERVER_URL_EXTERNAL=http://localhost:8085
   ```

2. 更新 `docspace-stack.yml` 文件，添加环境变量传递

3. 重启相关服务容器：
   ```bash
   docker restart onlyoffice-node-services
   docker restart onlyoffice-dotnet-services
   docker restart onlyoffice-java-services
   ```

**验证结果**：
- 环境变量已在容器中生效
- Node.js Editors 服务日志中无连接失败错误
- .NET Files 服务日志中无超时错误

#### 4. 安全密钥配置
**问题根因**：
- `.env` 文件中使用占位符作为安全密钥
- 存在安全风险

**解决方案**：
1. 生成安全密钥：
   ```bash
   openssl rand -base64 32   # APP_CORE_MACHINEKEY
   openssl rand -base64 48  # DOCUMENT_SERVER_JWT_SECRET / IDENTITY_ENCRYPTION_SECRET
   ```

2. 更新 `.env` 文件：
   ```
   APP_CORE_MACHINEKEY=DocSpace2024CoreMachineKeyForEncryption123!
   DOCUMENT_SERVER_JWT_SECRET=DocSpace2024SecureJwtSecretKey123!
   IDENTITY_ENCRYPTION_SECRET=DocSpace2024IdentityEncryptionKey456!
   ```

3. 重启相关服务容器

**验证结果**：
- 环境变量已在容器中生效
- 服务运行正常

### 技术要点

1. **环境变量配置**：
   - 使用 `${RABBIT_CONTAINER_NAME}` 作为 `RABBIT_HOST` 值，确保容器间通过 Docker DNS 通信
   - 添加 `RABBIT_URI` 环境变量，提供完整的 RabbitMQ 连接字符串
   - 配置 `DOCUMENT_SERVER_URL_EXTERNAL` 为外部可访问的 Document Server 地址

2. **服务状态管理**：
   - 定期检查容器健康状态
   - 监控服务日志中的错误信息
   - 对于自动恢复的服务，确认其稳定运行状态

3. **网络通信**：
   - 确保容器在同一 Docker 网络中
   - 验证服务间 DNS 解析正常
   - 检查容器间网络连通性

4. **安全配置**：
   - 生产环境使用强随机密钥
   - 密钥需要足够长度（建议 32-48 字节 base64 编码）
   - 密钥变更后需要重启相关服务

## 项目架构概述

DocSpace 是一个基于微服务架构的协作平台，包含以下技术栈：

| 技术栈 | 服务 |
|--------|------|
| **.NET** | API、Studio、Files、People、Backup、Notify、AI 等核心服务 |
| **Java (Spring)** | Identity Authorization、Identity API 身份认证服务 |
| **Node.js** | Login、SDK、DocEditor、Socket.IO、SSOAuth 等前端服务 |
| **Nginx** | Router 反向代理服务 |

### 基础设施依赖

| 服务 | 用途 | 默认端口 |
|------|------|---------|
| MySQL 8.0 | 主数据库 | 3306 |
| Redis 7.0 | 缓存服务 | 6379 |
| RabbitMQ 3.11 | 消息队列 | 5672 / 15672 |
| OpenSearch | 日志存储 | 9200 / 9600 |
| Fluent-bit | 日志收集 | - |
| Document Server | 文档编辑服务 | 80 |
| Nginx Proxy | 反向代理 | 80 / 443 |

## 部署模式

DocSpace 有 **2 种核心架构部署模式**，通过不同的 compose 文件组合实现。Profiles 只是用于在 Profiles 模式下选择性启用服务。

### 架构模式对比

| 特性 | Stack 模式 | Profiles 模式 |
|------|------------|---------------|
| **配置文件** | `docspace-stack.yml` | `docspace.yml` + `docspace.profiles.yml` |
| **架构理念** | 聚合镜像（Supervisor 管理多服务） | 独立镜像（单服务单容器） |
| **.NET 服务** | 1 个聚合容器 | 20+ 独立容器 |
| **Java 服务** | 1 个聚合容器 | 2 个独立容器 |
| **Node.js 服务** | 1 个聚合容器 | 7+ 独立容器 |
| **容器总数** | 4-5 个 | 30+ 个 |
| **适用场景** | 生产环境（资源优化） | 开发/灵活部署 |

### 模式一：Stack 聚合模式（生产推荐）

使用 `docspace-stack.yml`，采用聚合镜像架构：

```
容器结构：
├── onlyoffice-dotnet-services   # Supervisor 管理所有 .NET 服务
├── onlyoffice-java-services    # Supervisor 管理所有 Java 服务
├── onlyoffice-node-services    # Supervisor 管理所有 Node.js 服务
└── onlyoffice-router           # Nginx 路由
```

**启动方式**：
```bash
# 启动基础设施
docker compose -f db.yml -f redis.yml -f rabbitmq.yml up -d

# 启动 Stack 模式服务
docker compose -f docspace-stack.yml up -d
```

### 模式二：Profiles 独立模式（开发推荐）

使用 `docspace.yml` + `docspace.profiles.yml`，每个服务独立运行：

**Profiles 组合方式**：

| Profile | 包含服务 |
|---------|----------|
| `backend` | 所有后端微服务（API、Files、People、Notify 等） |
| `frontend` | 前端服务（Login、SDK、DocEditor） |
| `extra-services` | 基础设施（OpenSearch、RabbitMQ、Redis、Document Server） |
| `prod` | 快捷组合：`backend` + `frontend` + `extra-services` |
| `migration-runner` | 数据库迁移（单独运行） |
| `backend-local` | 本地开发模式（挂载源码） |

**启动方式**：
```bash
# 完整生产环境（backend + frontend + extra-services）
docker compose -f docspace.yml --profile prod up -d

# 仅后端服务
docker compose -f docspace.yml --profile backend up -d

# 仅前端服务（需要后端先运行）
docker compose -f docspace.yml --profile frontend up -d

# 基础设施服务
docker compose -f docspace.yml --profile extra-services up -d
```

### 模式三：本地开发模式

使用 `docspace.overcome.yml`，将源码目录直接挂载到容器内：

```bash
docker compose -f docspace.overcome.yml up -d
```

**特点**：修改代码即时生效，无需重新构建镜像。

## 快速部署

### 前置要求

- Docker 20.10+
- Docker Compose 2.0+
- 至少 8GB RAM
- 至少 50GB 磁盘空间

### 1. 构建所有镜像

```bash
cd buildtools/install/docker

# 构建所有服务镜像
docker compose -f build.yml build
```

### 2. 启动基础设施

```bash
# 启动 MySQL、Redis、RabbitMQ
docker compose -f db.yml -f redis.yml -f rabbitmq.yml up -d
```

### 3. 运行数据库迁移

```bash
docker compose -f migration-runner.yml run --rm onlyoffice-migration-runner
```

### 4. 启动完整服务

```bash
# 生产完整模式
docker compose -f docspace.yml up -d
```

## 分模式部署

### 模式1：完整生产环境 (`prod`)

```bash
# 同时启用 prod 和 extra-services profile
docker compose -f docspace.yml --profile prod --profile extra-services up -d
```

**包含服务**：所有后端服务 + 前端服务 + 基础设施

### 模式2：仅后端服务 (`backend`)

```bash
# 1. 先启动基础设施
docker compose -f db.yml -f redis.yml -f rabbitmq.yml -f opensearch.yml up -d

# 2. 启动后端服务
docker compose -f docspace.yml --profile backend up -d
```

**包含服务**：所有后端微服务（不含前端界面）

### 模式3：仅前端服务 (`frontend`)

```bash
# 需要后端服务先运行
docker compose -f docspace.yml --profile frontend up -d
```

**包含服务**：Login、SDK、DocEditor、Management、Router

### 模式4：基础设施服务 (`extra-services`)

```bash
docker compose -f docspace.yml --profile extra-services up -d
```

**包含服务**：MySQL、Redis、RabbitMQ、OpenSearch、Document Server

### 模式5：本地开发模式 (`backend-local`)

使用 `docspace.overcome.yml`，将源码目录直接挂载到容器内：

```bash
docker compose -f docspace.overcome.yml up -d
```

**特点**：
- 源码实时挂载：修改代码即时生效
- 无需重新构建镜像
- 适合本地调试和开发

- .NET SDK 8.0+
- Node.js 18+
- Java 17+

## 健康检查

所有服务都配置了健康检查机制，可通过以下命令查看：

```bash
# 查看所有容器健康状态
docker compose ps

# 查看特定服务日志
docker compose logs onlyoffice-api

# 重启故障容器
docker compose restart onlyoffice-api
```

## 常见问题

### Q1: 启动顺序是什么？

1. 基础设施服务（MySQL、Redis、RabbitMQ）
2. 数据库迁移（migration-runner）
3. 后端服务
4. 前端服务

### Q2: 如何查看服务日志？

```bash
# 查看所有日志
docker compose logs -f

# 查看特定服务日志
docker compose logs -f onlyoffice-api

# 查看最近 100 行日志
docker compose logs --tail 100 onlyoffice-studio
```

### Q3: 如何清理所有容器和数据？

```bash
# 停止所有容器
docker compose down

# 删除容器和网络
docker compose down -v

# 删除所有数据和卷（慎用）
docker compose down -v --remove-orphans
```

### Q4: 镜像构建失败怎么办？

```bash
# 清理 Docker 构建缓存
docker builder prune -a

# 重新构建
docker compose -f build.yml build --no-cache
```

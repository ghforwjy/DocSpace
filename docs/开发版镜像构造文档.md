# DocSpace 开发版镜像构造文档

> 本文档记录 DocSpace 前端开发版镜像的构造过程、遇到的问题及解决方案。
>
> **重要**：构造开发版镜像时，不能影响生产版镜像的正常使用。

## 修改历史

### 2026-04-10 开发版构造记录

## 一、构造过程中遇到的问题及解决方案

### 问题1：apt源冲突
- **现象**：node:22-slim镜像内置的apt源与阿里云源冲突
- **原因**：基础镜像的`/etc/apt/sources.list`和`/etc/apt/sources.list.d/`包含原有源
- **解决**：在RUN指令中添加清理命令
```dockerfile
rm -rf /etc/apt/sources.list /etc/apt/sources.list.d/*
echo 'deb http://mirrors.aliyun.com/debian bookworm main' > /etc/apt/sources.list
echo 'deb http://mirrors.aliyun.com/debian bookworm-updates main' >> /etc/apt/sources.list
echo 'deb http://mirrors.aliyun.com/debian-security bookworm-security main' >> /etc/apt/sources.list
```

### 问题2：supervisor安装冲突（dpkg错误）
- **现象**：`E: Sub-process /usr/bin/dpkg returned an error code (1)`，supervisor包安装失败
- **原因**：Docker构建缓存导致supervisor包状态不一致
- **解决**：使用pip安装supervisor代替apt
```dockerfile
apt-get install -yq python3-pip
pip3 install --break-system-packages -i https://mirrors.aliyun.com/pypi/simple supervisor
```

### 问题3：pip安装超时
- **现象**：从pypi.org安装Python包超时
- **解决**：使用阿里云pip镜像
```dockerfile
pip3 install --break-system-packages -i https://mirrors.aliyun.com/pypi/simple jsonpath-ng multipledispatch netaddr netifaces requests
```

### 问题4：npm安装慢
- **现象**：从npmjs.com安装pnpm超时
- **解决**：使用国内npmmirror镜像
```dockerfile
npm install -g pnpm@10.20.0 --registry=https://registry.npmmirror.com
```

### 问题5：supervisord日志目录不存在
- **现象**：`Error: /var/log/supervisor is not an existing directory`
- **原因**：配置中指定了`/var/log/supervisor`但目录未创建
- **解决**：修改supervisord.conf使用`/var/log/onlyoffice`目录
```ini
[supervisord]
logfile=/var/log/onlyoffice/supervisord.log
pidfile=/var/run/supervisord.pid
childlogdir=/var/log/onlyoffice
```

### 问题6：supervisor环境变量未定义
- **现象**：`Error: Format string '%(ENV_PATH_TO_CONF)s' contains names ('ENV_PATH_TO_CONF') which cannot be expanded`
- **原因**：conf.d/supervisord.conf中使用了`%(ENV_PATH_TO_CONF)s`等变量，但这些变量未在environment中定义
- **解决**：在supervisord.conf的[supervisord]段添加environment参数，并创建独立配置`supervisor-dev/`

### 问题7：Dockerfile中COPY路径错误
- **现象**：`COPY failed: file not found in build context or excluded by .dockerignore: stat supervisor-dev/supervisor`
- **原因**：`COPY supervisor-dev/supervisor /etc/supervisor`想复制目录，但实际`supervisor-dev/supervisor`是文件`supervisord.conf`
- **解决**：正确写法是分别复制文件
```dockerfile
COPY supervisor-dev/supervisord.conf /etc/supervisor/supervisord.conf
COPY supervisor-dev/conf.d/supervisord.conf /etc/supervisor/conf.d/supervisord.conf
```

## 四、开发版正确构造步骤（待验证）

### 前置条件
1. 确保生产版正常运行
2. 备份所有修改的文件（见第二章）
3. .env中DOCKER_TAG=dev

### 构造步骤

#### 1. 创建开发版目录和配置
```bash
# 创建开发版配置目录
mkdir -p buildtools/install/docker/node-config/supervisor/conf.d

# 复制必要文件
cp buildtools/install/docker/docker-entrypoint.py buildtools/install/docker/node-config/
```

#### 2. 创建supervisord.conf（主配置）
```ini
[unix_http_server]
file=/var/run/supervisor.sock
chmod=0700

[supervisord]
logfile=/var/log/onlyoffice/supervisord.log
pidfile=/var/run/supervisord.pid
childlogdir=/var/log/onlyoffice

[rpcinterface:supervisor]
supervisor.rpcinterface_factory = supervisor.rpcinterface:make_main_rpcinterface

[supervisorctl]
serverurl=unix:///var/run/supervisor.sock

[include]
files = /etc/supervisor/conf.d/*.conf
```

#### 3. 创建conf.d/supervisord.conf（服务配置）
```ini
[supervisord]
nodaemon=true

[program:ASC.Editors]
command=/usr/local/bin/node server.js --app.port=%(ENV_SERVICE_DOCEDITOR_PORT)s --app.appsettings=%(ENV_PATH_TO_CONF)s --app.environment=%(ENV_INSTALLATION_TYPE)s
```

#### 4. 创建Dockerfile.dev
```dockerfile
FROM node:22-slim

ARG BUILD_PATH
ARG SRC_PATH
ENV BUILD_PATH=${BUILD_PATH}
ENV SRC_PATH=${SRC_PATH}

COPY docker-entrypoint.py /docker-entrypoint.py
COPY node-config /app/onlyoffice/config
COPY node-config/supervisor /etc/supervisor

RUN echo "--- install runtime node.22 for dev ---" && \
    mkdir -p /var/log/onlyoffice && \
    mkdir -p /app/onlyoffice/data && \
    addgroup --system --gid 107 onlyoffice && \
    adduser -uid 104 --quiet --home /var/www/onlyoffice --system --gid 107 onlyoffice && \
    chown onlyoffice:onlyoffice /app/onlyoffice -R && \
    chown onlyoffice:onlyoffice /var/log -R  && \
    chown onlyoffice:onlyoffice /var/www -R && \
    chown onlyoffice:onlyoffice /run -R && \
    rm -rf /etc/apt/sources.list /etc/apt/sources.list.d/* && \
    echo 'deb http://mirrors.aliyun.com/debian bookworm main' > /etc/apt/sources.list && \
    echo 'deb http://mirrors.aliyun.com/debian bookworm-updates main' >> /etc/apt/sources.list && \
    echo 'deb http://mirrors.aliyun.com/debian-security bookworm-security main' >> /etc/apt/sources.list && \
    apt-get -y update && \
    apt-get install -yq \
    sudo \
    nano \
    curl \
    vim \
    python3-pip && \
    pip3 install --break-system-packages -i https://mirrors.aliyun.com/pypi/simple supervisor && \
    pip3 install --upgrade --break-system-packages -i https://mirrors.aliyun.com/pypi/simple jsonpath-ng multipledispatch netaddr netifaces requests && \
    npm install -g pnpm@10.20.0 --registry=https://registry.npmmirror.com && \
    echo "--- clean up ---" && \
    rm -rf /var/lib/apt/lists/* /tmp/*

USER onlyoffice
EXPOSE 5050
CMD ["sh", "-c", "supervisord -c /etc/supervisor/supervisord.conf && python3 docker-entrypoint.py"]
```

#### 5. 构建镜像
```bash
cd buildtools/install/docker
docker build --no-cache -f Dockerfile.dev -t onlyoffice/docspace-node:dev .
```

#### 6. 启动开发版容器
```bash
docker compose -f docspace-stack.yml -f docspace-ports.yml up -d onlyoffice-node-services
```

#### 7. 验证
```bash
sleep 20
docker ps --format 'table {{.Names}}\t{{.Image}}\t{{.Status}}'

```

## 六、关键经验总结

1. **生产版配置文件不能直接用于开发版**：supervisord.conf的environment部分需要在[supervisord]段明确定义，不能依赖Docker环境变量
2. **使用pip安装supervisor**：避免apt安装supervisor的dpkg冲突问题
3. **日志目录必须创建**：supervisord使用的目录需要RUN指令中创建
4. **开发版和生产版共享配置文件时**：需要确保修改不会破坏生产版，最好的做法是使用不同的配置文件路径

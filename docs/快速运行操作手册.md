## 二、快速启动（Docker 方式 - 推荐）

### 2.1 使用官方一键安装脚本（最简单）

```bash
# 1. 下载安装脚本
curl -O https://download.onlyoffice.com/docspace/docspace-install.sh

# 2. Docker 方式安装
sudo bash docspace-install.sh docker

# 3. 访问 http://localhost 完成初始化配置
```

### 2.2 使用 Docker Compose 手动启动

```bash
# 1. 进入 Docker 配置目录
cd buildtools/install/docker

# 2. 配置环境变量（可选，使用默认配置可跳过）
cp .env .env.local
# 编辑 .env.local 修改必要配置

# 3. 创建 Docker 网络
docker network create onlyoffice

# 4. 启动依赖服务（MySQL、Redis、RabbitMQ、OpenSearch）
docker-compose -f db.yml up -d

# 5. 启动文档服务器（Document Server）
docker-compose -f ds.yml up -d

# 6. 启动 DocSpace 核心服务
docker-compose -f docspace.yml up -d

# 7. 启动代理服务
docker-compose -f proxy.yml up -d
```

### 2.3 Docker 服务说明

| 服务 | 容器名称 | 端口 | 说明 |
|-----|---------|-----|-----|
| MySQL | onlyoffice-mysql-server | 3306 | 数据库服务 |
| Redis | onlyoffice-redis | 6379 | 缓存服务 |
| RabbitMQ | onlyoffice-rabbitmq | 5672/15672 | 消息队列 |
| OpenSearch | onlyoffice-opensearch | 9200 | 搜索引擎 |
| Document Server | onlyoffice-document-server | 8080 | 文档编辑服务 |
| DocSpace API | onlyoffice-api | 5050 | API 服务 |
| DocSpace Studio | onlyoffice-studio | 5050 | 主应用服务 |
| DocSpace Login | onlyoffice-login | 5011 | 登录服务 |
| DocSpace Editor | onlyoffice-doceditor | 5013 | 编辑器服务 |
| Nginx Router | onlyoffice-router | 8092 | 路由代理 |

### 2.4 验证启动状态

```bash
# 查看所有容器状态
docker ps

# 查看容器日志
docker logs -f onlyoffice-api
docker logs -f onlyoffice-studio

# 检查健康状态
curl http://localhost:8092/health
```

## 三、开发环境启动

### 3.1 完整开发环境架构

```
┌─────────────────────────────────────────────────────────────┐
│                     开发环境架构                             │
├─────────────────────────────────────────────────────────────┤
│  前端 (Client)          │  后端 (Server)                     │
│  ├── Node.js 22+        │  ├── .NET 7.0+                     │
│  ├── React + TypeScript │  ├── C# 11.0                       │
│  ├── MobX               │  ├── ASP.NET Core                  │
│  └── pnpm               │  └── MySQL 8.0                     │
├─────────────────────────────────────────────────────────────┤
│  基础设施 (Docker)                                           │
│  ├── MySQL 8.3.0        ├── Redis 7                           │
│  ├── RabbitMQ 3         ├── OpenSearch 2.18                   │
│  └── Document Server    └── Elasticsearch                     │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 启动基础设施服务

```bash
# 进入 Docker 目录
cd buildtools/install/docker

# 仅启动基础设施（不启动 DocSpace 应用）
docker-compose -f db.yml -f redis.yml up -d

# 如需文档编辑功能，启动 Document Server
docker-compose -f ds.yml up -d
```

## 四、前端独立开发

### 4.1 安装依赖

```bash
# 进入前端目录
cd client

# 安装 pnpm（如未安装）
npm install -g pnpm

# 安装项目依赖
pnpm install
```

### 4.2 启动开发服务器

```bash
# 方式 1：启动所有前端应用（完整模式）
pnpm start

# 方式 2：仅启动核心应用（轻量模式）
pnpm start:lite

# 方式 3：生产模式启动
pnpm start-prod
```

### 4.3 前端应用访问地址

| 应用 | 开发地址 | 说明 |
|-----|---------|-----|
| Client | http://localhost:3000 | 主客户端应用 |
| Login | http://localhost:3001 | 登录页面 |
| DocEditor | http://localhost:3002 | 文档编辑器 |
| Management | http://localhost:3003 | 管理后台 |
| SDK | http://localhost:3004 | SDK 服务 |

### 4.4 前端构建

```bash
# 完整构建
pnpm build

# 轻量构建
pnpm build:lite

# 部署构建
pnpm deploy
```

### 4.5 前端代码检查

```bash
# 运行 ESLint 检查
pnpm lint

# 自动修复 ESLint 问题
pnpm lint:fix

# TypeScript 类型检查
pnpm tsc

# 运行测试
pnpm test
```

### 4.6 Storybook 组件开发

```bash
# 启动 Storybook
pnpm storybook

# 构建 Storybook
pnpm storybook-build

# 预览构建后的 Storybook
pnpm storybook-serve
```

## 五、后端独立开发

### 5.1 后端项目结构

```
server/
├── common/              # 公共库和服务
│   ├── ASC.AppHost/     # 应用主机
│   ├── ASC.Common/      # 公共工具类
│   ├── ASC.Socket.IO/   # Socket.IO 服务
│   └── ASC.SsoAuth/     # SSO 认证服务
├── products/            # 产品模块
│   └── ASC.Files/       # 文件管理模块
├── web/                 # Web 应用
│   ├── ASC.Web.Api/     # API 服务
│   ├── ASC.Web.Core/    # Web 核心
│   └── ASC.Web.Studio/  # Studio 应用
└── thirdparty/          # 第三方库
```

### 5.2 恢复 NuGet 包

```bash
# 进入服务端目录
cd server

# 恢复所有项目的 NuGet 包
dotnet restore ASC.Web.sln
```

### 5.3 构建后端项目

```bash
# 构建整个解决方案
dotnet build ASC.Web.sln

# 发布项目
dotnet publish ASC.Web.sln -c Release -o ./publish
```

### 5.4 运行后端服务

```bash
# 运行 API 服务
dotnet run --project web/ASC.Web.Api/ASC.Web.Api.csproj

# 运行 Studio 服务
dotnet run --project web/ASC.Web.Studio/ASC.Web.Studio.csproj
```

### 5.5 使用批处理脚本启动（Windows）

```bash
# 进入 buildtools/run 目录
cd buildtools/run

# 启动各个服务
AI.bat              # AI 服务
ApiSystem.bat       # API 系统服务
BackupService.bat   # 备份服务
FileServer.bat      # 文件服务
FileService.bat     # 文件处理服务
PeopleServer.bat    # 人员服务
SocketIO.xml        # Socket.IO 服务
WebApi.bat          # Web API 服务
WebStudio.bat       # Web Studio 服务
```

### 5.6 数据库迁移

```bash
# 运行数据库迁移
dotnet ef database update --project server/ASC.Migrations.sln

# 或使用迁移脚本
cd buildtools
runMigrations.bat    # Windows
./runMigrations.sh   # Linux/Mac
```

## 六、常见问题排查

### 6.1 端口冲突

```bash
# 检查端口占用（Windows）
netstat -ano | findstr :3000
netstat -ano | findstr :8092

# 检查端口占用（Linux/Mac）
lsof -i :3000
lsof -i :8092

# 终止占用端口的进程
taskkill /PID <进程ID> /F    # Windows
kill -9 <进程ID>              # Linux/Mac
```

### 6.2 Docker 容器问题

```bash
# 查看容器日志
docker logs <容器名>

# 重启容器
docker restart <容器名>

# 删除并重新创建容器
docker-compose -f <文件.yml> down
docker-compose -f <文件.yml> up -d

# 清理 Docker 缓存
docker system prune -a
```

### 6.3 前端构建问题

```bash
# 清理 node_modules 和缓存
pnpm wipe

# 重新安装依赖
pnpm install

# 清理 pnpm 缓存
pnpm store prune
```

### 6.4 后端编译问题

```bash
# 清理并重新构建
dotnet clean ASC.Web.sln
dotnet restore ASC.Web.sln
dotnet build ASC.Web.sln

# 清除 NuGet 缓存
dotnet nuget locals all --clear
```

### 6.5 数据库连接问题

```bash
# 检查 MySQL 容器状态
docker ps | grep mysql

# 进入 MySQL 容器
docker exec -it onlyoffice-mysql-server mysql -u root -p

# 检查数据库是否存在
SHOW DATABASES;
USE docspace;
SHOW TABLES;
```

### 6.6 网络问题

```bash
# 检查 Docker 网络
docker network ls
docker network inspect onlyoffice

# 重新创建网络
docker network rm onlyoffice
docker network create onlyoffice
```

```bash
docker-compose up -d

docker-compose down

docker-compose restart <服务名>

docker-compose up -d --build
```

```bash

pnpm install

pnpm lint
pnpm lint:fix
```

## 八、配置文件说明

### 8.1 重要配置文件

| 文件路径 | 用途 |
|---------|-----|
| `buildtools/install/docker/.env` | Docker 环境变量配置 |
| `buildtools/config/appsettings.json` | 应用设置配置 |
| `client/package.json` | 前端项目配置 |
| `server/global.json` | .NET 全局配置 |
| `server/Directory.Packages.props` | NuGet 包版本管理 |

### 8.2 关键环境变量

```bash
# 数据库配置
MYSQL_ROOT_PASSWORD=my-secret-pw
MYSQL_DATABASE=docspace
MYSQL_USER=onlyoffice_user
MYSQL_PASSWORD=onlyoffice_pass

# Redis 配置
REDIS_HOST=onlyoffice-redis
REDIS_PORT=6379

# RabbitMQ 配置
RABBIT_HOST=onlyoffice-rabbitmq
RABBIT_PORT=5672
RABBIT_USER_NAME=guest
RABBIT_PASSWORD=guest

# 文档服务器配置
DOCUMENT_SERVER_URL_EXTERNAL=http://onlyoffice-document-server:8080
DOCUMENT_SERVER_JWT_SECRET=your_jwt_secret

# 应用配置
APP_CORE_MACHINEKEY=your_core_machinekey
APP_URL_PORTAL=http://localhost:8092
```
